这份文档是什么？ 记录了把 Claude Code(命令行版) 接到第三方硅基流动 API 的完整配置流程， 以及一个自制的「无 GUI 多供应商切换器 ccx」的用法。它的作用等价于 cc-switch 对 Claude Code 做的事—— 多家大模型、一键切换——但完全在终端里完成，不需要图形界面，也不需要 root 权限。

0为什么没用 cc-switch 图形版

cc-switch 是 Tauri 桌面 GUI 应用。安装本身已成功（deb 及 libwebkit2gtk-4.1 等依赖都装好了）， 但运行时崩溃：Failed to initialize GTK。

根因：这台机器停在 GDM 登录界面，没有任何用户登录进 GNOME 桌面；Xorg/gnome-shell 都属于 gdm， 而 xx03 只是通过 SSH 连入，没有可绘制的图形显示，因此 GUI 无法启动。

要用 GUI 版：需在机器物理屏幕前用 xx03 登录进桌面，或配置 VNC/远程桌面，再在桌面终端运行 cc-switch。 纯 SSH 场景下，下面的 CLI 方案是更合适的等效替代。

1原理：Claude Code 如何接第三方 API

Claude Code 走 Anthropic Messages API 协议，通过下面几个环境变量即可指向任意兼容端点：

环境变量作用本方案取值
ANTHROPIC_BASE_URLAPI 地址（不带 /v1，CC 会自动补 /v1/messages）https://api.siliconflow.cn
ANTHROPIC_AUTH_TOKENBearer 鉴权 token（首选）你的硅基流动 Key
ANTHROPIC_MODEL主模型 IDzai-org/GLM-5.2
ANTHROPIC_SMALL_FAST_MODEL后台小任务模型zai-org/GLM-5.2
硅基流动官方已提供 Anthropic 兼容端点，所以无需 OneAPI/LiteLLM 之类的协议转换代理，直连即可。 zai-org/GLM-5.2 为已核实的模型 ID。

2我已经帮你做好的部分

  • 确认环境：Ubuntu 22.04、claude CLI 2.1.152、claude 以 xx03 身份运行、~/.claude 可写。
  • 核实硅基流动 Anthropic 兼容端点与 zai-org/GLM-5.2 模型 ID。
  • 创建无 GUI 切换器脚本：~/.config/claude-switch/ccx.sh
  • 创建凭据文件（占位，权限 600）：~/.config/claude-switch/keys.env
  • 挂载到 ~/.bashrc（幂等，新终端自动加载）。
  • 功能自测通过（ccx list / glm / status / official 均正常，测试后已复位为 official，不影响当前登录）。

3你现在要做的两步 关键

① 填入硅基流动 API Key（一次性）

nano ~/.config/claude-switch/keys.env

把这一行的占位值换成你真实的 Key（以 sk- 开头）：

SILICONFLOW_KEY="sk-你的真实硅基流动Key"
# glm / deepseek / kimi 共用这一个 Key

② 新开一个终端，切换并启动

ccx glm        # 切到硅基流动 GLM-5.2
claude         # 启动，即用 GLM-5.2
务必新开终端（让 ~/.bashrc 生效）。你当前正在跑的 claude 进程不会自动切换，需退出后重启。

4日常用法

编辑 keys.env 填 Key（仅一次）→ ccx <供应商>→ claude
命令说明
ccx list列出所有供应商 + 当前选择
ccx glm硅基流动 GLM-5.2 （已核实可用）
ccx glmv硅基流动 GLM-4.5V 多模态（看图）
ccx multi多槽位映射：进 claude 后用 /model 实时切换多个模型 推荐
ccx deepseek硅基流动 DeepSeek（模型 slug 见第 6 节说明）
ccx kimi硅基流动 Kimi（模型 slug 见第 6 节说明）
ccx official切回官方 Claude 登录（OAuth）
ccx status查看当前 BASE_URL / MODEL / 槽位映射
新开的终端会自动记住你上次选的供应商（记录在 ~/.config/claude-switch/current）。每次切换后重启 claude 即生效。

★ 会话内自由切换多个模型（多槽位映射 + 多模态）

Claude Code 运行时会多模型协作：主推理模型 + 背景小任务模型。它支持把 Opus/Sonnet/Haiku 三个「槽位」各自映射到不同模型， 并可给每个槽位声明能力（如 vision 视觉）。用 ccx multi 一次配好后，进入 claude 用 /model 命令即可 在同一会话里实时切换，无需退出重启。

Claude 槽位映射的硅基流动模型用途能力
Opuszai-org/GLM-5.2旗舰编码 / 长程推理tools, thinking
Sonnetzai-org/GLM-4.5V多模态 / 看图 / GUI / 文档vision, tools
Haikuzai-org/GLM-4.1V-9B-Thinking便宜快速 / 背景任务（也能看图）vision, tools
自定义项deepseek-ai/DeepSeek-V4-Pro旗舰 / 深度推理（无视觉）tools, thinking
ccx multi      # 配好多槽位映射
claude         # 启动
# 进入会话后：
/model         # 弹出菜单，在 GLM-5.2 / GLM-4.5V / GLM-4.1V / DeepSeek-V4-Pro 间实时切换
多模态说明：需要看图/截图分析时，用 /model 切到 Sonnet(GLM-4.5V)——标了 vision 能力，可直接接收图片输入。 需要更强的纯代码/深度推理时切到自定义 DeepSeek-V4-Pro（无视觉）；日常纯代码切回 Opus(GLM-5.2)。背景小任务由 Haiku(GLM-4.1V) 承担，省钱。

5验证 Key 是否可用

填好 Key 后，测试连通性（能列出模型即 Key 正常）：

source ~/.config/claude-switch/keys.env
curl -s -H "Authorization: Bearer $SILICONFLOW_KEY" \
     https://api.siliconflow.cn/v1/models | head -c 300; echo

6自定义 · 添加更多模型

编辑 ~/.config/claude-switch/ccx.sh，在 _ccx_apply() 的 case 里照葫芦画瓢加一段即可。例如加通义千问：

qwen)
  export ANTHROPIC_BASE_URL="https://api.siliconflow.cn"
  export ANTHROPIC_AUTH_TOKEN="$SILICONFLOW_KEY"
  export ANTHROPIC_MODEL="Qwen/Qwen3-235B-A22B"
  export ANTHROPIC_SMALL_FAST_MODEL="Qwen/Qwen3-235B-A22B" ;;

然后把 qwen 也加进 ccx() 里那行 glm|deepseek|kimi|official) 的匹配即可。

模型 ID 提醒：deepseek / kimi / qwen 里的 slug 为示例值。若调用报 model not found， 请到 硅基流动模型页 复制准确的模型 ID 替换。zai-org/GLM-5.2 已核实无误。

7还原 / 卸载

  • 临时切回官方：ccx official 后重启 claude。
  • 彻底移除切换器：删掉 ~/.bashrc 里 >>> claude-switch >>> 与 <<< claude-switch <<< 之间的三行， 再删除目录 ~/.config/claude-switch/。
  • 清理下载的安装包：rm -rf "/home/xx03/骨关节炎_FINAL/.cc-switch-dl"（CLI 方案用不到）。
